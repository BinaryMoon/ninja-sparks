Two weeks have flown by and it's time for a new issue of Ninja Sparks. It feels a bit strange writing this considering everything happening in the world at the moment, but I want to maintain a small sense of normality, so I will keep doing my thing as much as I can. I hope you and your loved ones are doing well.

Over the last couple of weeks I’ve been working on two main things: small updates to Brush Ninja and a larger update to finish my game [Splatter Balls](https://brush.ninja/play/splatter-balls/).

At the encouragement of a couple of friends, I’ve decided to try releasing it on some game portals. So I’ve been adding new obstacles and levels, tidying up the logic, and adding music and sound effects.

One of the things I particularly enjoy is that I made a level editor for the game, so my son has been helping me create levels. He’s enjoyed building them and even coming up with ideas when he’s not on the computer, which I really want to encourage. Not because it will make the game bigger, but because I want him to experience the joy of creating things the way I do.

In addition, I’m now wondering if I could make the editor public and add a way for people to submit levels for possible inclusion in the game.

## Drawing Timer

I said at the start of the year that I want to get better at art, and I’ve found a new way to practise. Practising drawing through timed exercises is a common technique, and I linked to some resources for that previously. However, most of those resources are themed around figure drawing, and I want to practise more specific types of art.

{% include embed-image.html image="/images/updates/2026/drawing-timer-settings.png" caption="Drawing Timer Settings" %}

Then last week I found [Drawing Timer](https://drawingtimer.com/), a free web app that lets you set up your own drawing exercises using Pinterest boards. You can enter the name of a board you’ve compiled, set the drawing times, and it will automatically switch to the next image when the time is up. It’s a really simple idea, but it’s perfect for what I want to do.

{% include embed-image.html image="/images/updates/2026/drawing-timer-practice.png" caption="Drawing Timer in action" %}

I’m now considering making a simple version of this that works with local images so I can use it for my own practice based on the library of images I’ve collected over the years.

**[Try Drawing Timer](https://drawingtimer.com/)**

## Gorillaz - The Mountain, The Moon Cave and The Sad God

{% include embed-youtube.html video="ucRulNQsuYQ" %}

A lovely new Gorillaz song and video. The video is the important part because it was hand-animated using traditional techniques, something you rarely see these days.

There is a behind-the-scenes “making of” video, but it’s premium so you can’t watch it without subscribing. The animation was created by The Line Animation Studio and it’s beautiful to see this sort of work being made. Especially at a time when AI-generated slop seems to be taking over so much of the visual arts.

I'm not a huge Gorillaz fan, but I like that they are putting art, craft, and creativity at the heart of their work and I hope this continues in the future.

[Watch the video](https://www.youtube.com/watch?v=ucRulNQsuYQ)

## PicoCAD 2 Feature Trailer

{% include embed-youtube.html video="tr-mW9gcito" youtube_thumbnail="https://i.ytimg.com/vi/tr-mW9gcito/0.jpg" %}

I've mentioned [PicoCAD]({% link _library/app/2025-02-10-picocad.md %}) before and now we have the trailer for version 2.

The new version includes a host of improvements. While the original app was made with Pico-8, the new version is built with Love2D, which is a more powerful system that allows more flexibility and better performance.

New features include a built-in texture editor, a new animation system, and object-based scenes that allow more complex models. It also has a new palette system that lets you use colours beyond the limited Pico-8 palette.

[Watch the video](https://www.youtube.com/watch?v=tr-mW9gcito)

## Sparked Discoveries

Here’s a handful of links I found intriguing recently:

### [DOOM on an Oscilloscope](https://front-end.social/@html5test/115967227563353825)

{% include embed-browsershots.html url="https://front-end.social/@html5test/115967227563353825" %}

Developers love putting DOOM on anything with a screen, but this is a new one. Niels Leenheer has managed to get [Doom running on an oscilloscope](https://front-end.social/@html5test/115967227563353825), a device normally used to display electrical signals.

It’s an impressive feat of engineering and programming. The Mastodon post includes both a video and a write-up explaining how it works. If you're into programming it’s worth reading for the technical details. If you're not, it’s still worth seeing just for the result.

<!-- INCLUDE SHARE -->

### [Brickset Lego Catalogues](https://brickset.com/library/catalogues)

{% include embed-image.html image="/images/updates/2026/brickset.png" %}

I’ve used Brickset for years, but for some reason I never noticed they have a whole section of scanned Lego catalogues. It’s really interesting to see how the sets have evolved over the years and it’s a great resource for anyone interested in Lego history. Plus I even recognised some of the catalogues from my childhood.

### [The Scale of Life](https://www.thescaleoflife.com/)

{% include embed-browsershots.html url="https://www.thescaleoflife.com/" %}

It’s a simple website, but there’s something about it that really puts things into perspective. The site notes the moment you arrive and then shows everything that has happened since then. In the time it took me to write this sentence, for example, there have been 158 births.

The first page is interesting, but make sure to explore the different categories because there’s a lot of data to compare. It’s a reminder that we’re all part of a much bigger picture and that our time here is just a small blip in the grand scheme of things.
